# 低龄宝宝视听绘本系统详细设计文档 v1.0

## 1. 项目概述

本系统是一个面向低龄宝宝家庭场景的移动端内容播放网站，主要服务于家长和老人陪伴宝宝时快速选择、播放音频和视频内容的需求。

系统一期聚焦最小可用版本，优先实现以下能力：

- 音频、视频资源浏览
- 搜索、筛选、排序
- 音频和视频播放
- 点击量统计
- 家长匿名评分
- 隐藏式管理入口
- 基础资源维护
- 路由插件内网穿透资源接入方案验证

绘本阅读功能作为二期内容，当前不纳入开发范围。

## 2. 产品定位

- 产品形态：手机端 H5 内容播放网站
- 主要用户：家长、老人
- 使用方式：无需登录，打开页面即可使用
- 管理方式：通过前台隐藏入口进入管理页
- 当前目标：快速搭建可用原型，优先验证资源链接方案是否稳定

## 3. 设计原则

- 优先简单可用，不做过度设计
- 前台操作尽量直观，方便老人使用
- 支持宝宝风格视觉，但整体交互应兼顾家长审美
- 一期不追求复杂权限体系，优先实现内容可播放、可维护
- 资源接入方案优先验证基于路由插件内网穿透映射出的资源 URL 是否稳定可用

## 4. 一期范围

### 4.1 一期包含

- 首页展示音频和视频资源
- 支持按关键字搜索
- 支持按年龄、系列、类型筛选

- 视频全页播放
- 点击次数统计
- 匿名 1-5 星评分

- 管理页基础维护：
  - 新增资源
  - 编辑资源
  - 上下架资源
  - 标记异常资源
- 系列字典维护
- 手动录入第三方资源链接

### 4.2 一期不包含

- 绘本阅读

## 5. 用户角色与使用场景

### 5.1 用户角色

- 普通用户：家长、老人
- 管理员：资源维护人员

### 5.2 典型使用场景

- 家长打开首页，搜索某个系列内容给宝宝播放
- 老人根据封面和昵称快速识别内容，例如“滴滴”
- 用户按年龄段筛选适合当前宝宝的内容
- 用户点击音频卡片，底部弹出播放器进行播放
- 用户点击视频卡片，进入全页视频播放
- 播放后用户对内容进行简单评分
- 管理员通过隐藏入口进入管理页，维护资源链接和上下架状态

系统主要包含两个部分：

- 前台用户侧

前台用户侧页面结构：

- 首页
- 视频播放页
- 音频底部播放器
- 搜索/筛选/排序交互区

管理侧页面结构：

- 管理首页/资源列表页
- 新增资源页
- 编辑资源页

## 7. 前台详细设计

### 7.1 首页设计

首页是系统核心页面，不设置推荐区，默认直接展示全部可用资源。

页面结构建议如下：

- 顶部区域
  - 页面标题
  - 搜索框
- 筛选区域
  - 系列筛选
  - 年龄筛选
  - 类型筛选
- 排序区域
  - 按评分排序
  - 按播放量排序
  - 按添加时间排序
- 内容区域
  - 音频区
  - 视频区
- 底部区域
  - 音频播放器抽屉
  - 隐藏管理入口触发区域

### 7.2 内容分区规则

- 默认音频在前，视频在后
- 后续可支持切换顺序，但一期默认固定即可
- 音频区和视频区都支持无限下拉加载

### 7.3 资源卡片设计

每条资源卡片建议包含：

- 封面图
- 展示名称
- 别名
- 系列
- 年龄段
- 播放量
- 平均评分

展示名称规则：

- 若配置了昵称，则优先显示昵称
- 若未配置昵称，则显示正式标题

昵称用于帮助老人和孩子快速识别内容，例如：

- 正式标题：`托马斯和朋友们第3集`
- 昵称：`滴滴`

则前台优先展示 `滴滴`。

### 7.4 搜索设计

支持全文关键字搜索，搜索字段包括：

- 标题
- 昵称
- 别名
- 系列名称

一期建议采用简单实现方式：

- 输入关键字后点击搜索按钮触发
- 或输入时即时搜索，二选一即可

如果优先简单开发，推荐先做“按钮触发搜索”。

### 7.5 筛选设计

筛选项如下：

- 年龄段
  - 全部
  - 0-2岁
  - 2-4岁
  - 4-6岁
- 类型
  - 全部
  - 音频
  - 视频
- 系列
  - 全部
  - 从系列字典读取可选项

一期默认筛选状态为“全部”。

### 7.6 排序设计

排序字段包括：

- 评分
- 播放量
- 添加时间

默认排序建议：

- 添加时间倒序

可切换为：

- 评分降序
- 播放量降序

## 8. 播放功能详细设计

### 8.1 音频播放设计

交互要求：

- 点击音频卡片后，计入一次点击
- 页面底部弹出音频播放器抽屉
- 自动开始播放当前音频

播放器建议包含：

- 封面
- 当前播放名称
- 播放/暂停按钮
- 播放进度
- 关闭按钮

一期要求：

- 支持基础播放
- 尽量支持后台持续播放
- 尽量支持锁屏播放

兼容性说明：

- H5 页面在不同手机浏览器下，对后台播放和锁屏播放支持不稳定
- 一期应将其定义为“尽量支持”，不是硬性必须完全稳定
- 若后续将其作为强需求，需考虑 PWA、小程序或 App 方案

### 8.2 视频播放设计

交互要求：

- 点击视频卡片后，计入一次点击
- 跳转到全页视频播放页面

播放器能力：

- 播放/暂停
- 进度条
- 全屏
- 封面展示

实现建议：

- 优先使用浏览器/设备默认视频播放能力
- 避免一期自研复杂控制层

## 9. 评分与统计设计

### 9.1 播放统计

统计方式：

- 点击资源即记一次播放量
- 不做严格去重
- 不要求用户登录
- 不要求设备级唯一校验

一期统计目标：

- 能够展示资源热度
- 为排序提供基础数据

### 9.2 评分设计

评分方式：

- 1-5 星
- 匿名评分
- 不做强校验
- 不限制重复评分

统计结果：

- 平均评分
- 评分次数

交互建议：

- 一期不强引导
- 可在播放页底部保留评分入口
- 或播放结束后轻量弹出评分提示

## 10. 隐藏管理入口设计

隐藏入口规则：

- 在页面底部空白区域连续点击 5 次
- 进入管理页
- 一期不需要密码
- 一期不需要登录

说明：

- 该方案仅适合当前内部试验阶段
- 若未来对外使用，需补充登录和权限控制

## 11. 管理端详细设计

### 11.1 管理页目标

管理页用于维护资源数据，不面向普通用户。

一期只提供基础维护能力：

- 查看资源列表
- 新增资源
- 编辑资源
- 上下架资源
- 标记异常
- 维护系列字典

### 11.2 管理页列表设计

列表展示字段建议：

- 标题
- 昵称
- 类型
- 系列
- 年龄段
- 上架状态
- 异常状态
- 添加时间
- 点击次数
- 平均评分

操作项：

- 编辑
- 上架/下架
- 标记异常/取消异常

前台展示规则：

- 仅展示 `上架=true 且 异常=false` 的资源

### 11.3 新增/编辑资源设计

建议字段如下：

基础字段：

- 标题
- 昵称
- 别名
- 系列
- 适合年龄
- 资源类型
- 资源链接
- 封面图
- 简介

状态与管理字段：

- 是否上架
- 是否异常
- 异常原因
- 来源备注
- 排序权重

建议辅助功能：

- 测试资源链接
- 预览封面图

其中：

- 系列从字典选择
- 年龄段从固定枚举中选择
- 类型固定为音频/视频二选一

系列不允许录入时自由输入，需从字典中选择。

原因：

- 避免同一系列名称录入不统一
- 方便后续筛选和数据归类
- 降低管理成本

系列字典字段建议：

- 系列名称
- 别名
- 启用状态
- 排序权重

## 13. 前后端接口设计

### 13.1 前台接口

`GET /api/media/list`

- 获取前台资源列表
- 支持参数：
  - keyword
  - mediaType
  - ageRange
  - seriesId
  - sortBy
  - order
  - page
  - pageSize
- 只返回上架且非异常资源

`GET /api/media/{id}`

- 获取单个资源信息
- 供播放页使用

`POST /api/media/{id}/click`

- 记录点击次数

`POST /api/media/{id}/rate`

- 提交评分
- 参数：
  - score

`GET /api/series/options`

- 获取前台筛选所需系列列表

### 13.2 管理接口

`GET /api/admin/media/list`

- 获取资源管理列表

`POST /api/admin/media`

- 新增资源

`PUT /api/admin/media/{id}`

- 编辑资源

`PATCH /api/admin/media/{id}/publish`

- 上下架资源

`PATCH /api/admin/media/{id}/abnormal`

- 标记或取消异常

`GET /api/admin/series/list`

- 获取系列字典列表

`POST /api/admin/series`

- 新增系列

`PUT /api/admin/series/{id}`

- 编辑系列

## 14. 资源接入方案详细设计

这部分是当前项目最关键的前置验证项。

### 14.1 目标

在不自建复杂媒体存储体系的前提下，通过路由器插件提供的内网穿透能力，将 USB 硬盘中的媒体资源映射为可访问 URL，并满足一期原型需要。

### 14.2 当前资源方案

#### 方案 A：路由插件内网穿透映射资源 URL

- 媒体资源存放在路由器连接的 USB 硬盘中
- 通过路由器插件将目录或文件映射为可访问 URL
- 系统在管理页中录入播放地址和封面地址
- 前台页面直接使用配置的 URL 播放音视频和展示封面

优点：

- 不需要额外建设媒体存储服务
- 资源由家庭现有硬盘统一管理
- 接入方式直接，符合当前使用场景
- 系统侧只需维护 URL 配置，开发成本较低

缺点：

- 资源可用性依赖路由器插件和家庭网络环境
- URL 稳定性需要实际验证
- 不适合作为未来正式对外产品的长期方案

建议：

- 作为一期唯一资源接入方案使用

#### 方案 B：录入第三方页面地址，再手动提取真实播放地址

- 先找到资源页
- 再人工转成可播放直链

优点：

- 可覆盖更多来源

缺点：

- 人工维护成本高
- 地址规则变化后容易失效
- 不适合作为长期稳定方案

建议：

- 可作为辅助方式，不建议作为主方案

#### 方案 C：本地下载后上传网盘/第三方存储，再使用外链

- 你先手工下载资源
- 上传到网盘
- 前台使用网盘外链播放

优点：

- 资源比第三方页面更可控

缺点：

- 很多网盘不支持稳定在线播放
- 容易限流
- 直链可能变化
- 不适合正式产品长期使用

建议：

- 仅做试验，不建议作为长期主方案

#### 方案 D：对象存储 + CDN

- 使用阿里云 OSS、腾讯云 COS、七牛云等
- 资源存储在对象存储中，通过 CDN 分发

优点：

- 最稳定
- 最专业
- 支持长期产品化
- 封面和媒体资源都可统一管理

缺点：

- 需要前期配置
- 有一定存储和流量成本

建议：

- 作为正式版本长期方案

### 14.3 当前建议路线

当前阶段采用单阶段策略：

- 通过路由插件内网穿透方式映射 USB 硬盘资源
- 在系统中手动配置对应资源 URL
- 优先验证播放稳定性、访问速度和浏览器兼容性

当前不考虑：

- 第三方资源直链
- 网盘直链
- 对象存储 + CDN

### 14.4 建议增加的资源校验字段

为方便后期排查资源问题，建议在资源表增加：

- `source_type`
- `source_platform`
- `last_checked_at`
- `check_status`
- `failure_reason`

### 14.5 资源验证实验方案

建议先准备 10 条测试样本：

- 5 条音频
- 5 条视频

每条资源验证以下内容：

- 是否能在手机浏览器直接播放
- 内网穿透映射 URL 是否稳定
- 是否几天后仍可访问
- 音频在后台或锁屏下是否继续播放
- 视频封面是否可正常显示
- 加载速度是否可接受
- 不同浏览器兼容性是否能接受

建议优先测试浏览器：

- 微信内置浏览器
- Android Chrome
- iPhone Safari

## 15. 非功能设计

### 15.1 性能

- 首页支持分页加载或无限滚动
- 封面图需压缩，避免首屏过重
- 音视频资源通过路由插件映射 URL 加载，页面本身不承载媒体存储压力

### 15.2 兼容性

目标兼容：

- iPhone Safari
- Android Chrome
- 常见国产手机浏览器
- 微信内置浏览器

风险点：

- 音频自动播放限制
- 后台持续播放限制
- 锁屏播放限制
- 路由器插件映射 URL 的稳定性

### 15.3 安全性

一期仅用于内部试验，可暂不做严格安全体系。

但需注意：

- 外部链接不要允许执行脚本内容
- 管理入口虽然隐藏，但仍建议避免暴露明显管理路径
- 后续正式上线必须补充登录鉴权和接口权限控制

### 15.4 可维护性

- 系列统一通过字典维护
- 资源状态有明确区分：上架、下架、异常
- 建议保留来源备注和校验信息，方便替换失效链接

## 16. 技术实现建议

前端建议重点能力：

- 移动端适配
- 组件化资源卡片
- 音频播放器抽屉
- 视频全页播放页
- 搜索/筛选/排序统一状态管理

后端建议重点能力：

- 简单资源管理接口
- 字典管理接口
- 点击统计和评分接口
- 资源状态过滤逻辑

数据库建议：

- 使用 PostgreSQL
- 以资源表为核心
- 系列表作为独立字典表
- 评分表独立
- 点击日志可先简化，后续再增强

## 17. 开发顺序建议

建议按以下顺序开发：

1. 先验证路由插件内网穿透资源方案
