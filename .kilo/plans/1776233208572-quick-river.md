# BB-Play 二期功能实施计划（Redis版）

## 一、项目现状分析

### 技术栈
- **后端**：Spring Boot 3.4.4 + MyBatis Plus 3.5.8 + PostgreSQL
- **前端**：Vue 3.5.13 + Vant 4.9.18 + TypeScript
- **认证**：JWT（管理端）+ device_id（匿名评分）
- **Redis**：❌ 未配置

### 现有功能
- 资源管理（音频/视频）
- 系列管理
- 评分系统
- 管理员认证

### 数据库表
- `media_resource`：资源主表
- `series_dict`：系列字典
- `media_rating`：评分记录
- `sys_user`：管理员用户

## 二、实施范围

### 核心功能
1. **统一身份与存储方案**：Redis + Cookie 轻量用户标识
2. **收藏功能**：收藏/取消收藏、收藏列表
3. **最近播放**：播放历史记录（上限30条）
4. **资源链接测试**：管理页链接可用性检测
5. **音频播放列表**：播放模式（单曲循环/列表循环/列表随机）
6. **前后端校验**：长度与必填一致性校验

### 非范围
- 视频播放模式扩展
- 跨设备账号体系
- 推荐算法

## 四、技术约束

### 编码规范
- SQL 写在 Mapper 中，使用 MyBatis Plus 规范
- 不在 XML 中写 SQL，不使用 `@Sql`
- 关键业务步骤补充简洁中文注释
- 遵循现有代码风格与目录结构

### 测试规范
- 后端单元测试超时时间固定为 60 秒
- 前端类型检查仅针对已修改文件执行

### 安全规范
- Cookie 使用 `HttpOnly` 标记
- Redis 键使用统一前缀避免冲突
- 链接测试设置合理超时避免阻塞

## 五、验收标准

### 功能验收
- [ ] 收藏功能：刷新页面后状态一致，重复收藏不重复，可直接进入播放页
- [ ] 最近播放：上限 30 条，重复播放仅更新时间和进度
- [ ] 链接测试：10 秒内返回结果，失效链接明确展示失败原因
- [ ] 音频播放：刷新后保留模式和位置，随机模式单轮不重复，视频页不显示音频控件
- [ ] 前后端校验：超长禁止输入，必填项显示 `*`，后端返回明确错误信息

### 性能验收
- [ ] Redis 操作响应时间 < 100ms
- [ ] 链接测试超时控制在 10 秒内
- [ ] 收藏列表分页加载流畅

### 安全验收
- [ ] Cookie 正确设置 `HttpOnly`
- [ ] Redis 键正确设置 TTL
- [ ] 链接测试不泄露敏感信息

## 六、风险与依赖

### 技术风险
- Redis 未配置：需要先搭建 Redis 环境
- 链接测试可能遇到跨域或防盗链限制
- 随机播放洗牌袋逻辑需要仔细测试

### 外部依赖
- Redis 服务器（开发/测试/生产环境）
- 网络访问权限（链接测试）

### 缓解措施
- 提供 Docker Compose 快速启动 Redis
- 链接测试添加 User-Agent 和 Referer 头
- 洗牌袋逻辑编写单元测试覆盖边界情况
