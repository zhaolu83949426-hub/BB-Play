# 三期功能详细设计-绘本阅读

## 1. 文档定位

本文档用于定义低龄宝宝视听绘本系统三期功能的详细设计，重点实现绘本阅读能力，让宝宝可以通过图片和语音朗读的方式体验绘本内容。

本次设计是在一期、二期能力基础上的增量扩展，默认沿用当前项目的技术结构与分层方式：

- 前端：Vue 3 + Vant 移动端页面
- 后端：Spring Boot + Spring Security + MyBatis Plus
- 数据库：PostgreSQL
- 状态存储：Redis
- 新增能力：文字转语音（TTS）

三期聚焦绘本阅读的核心体验，提供自动朗读、自动翻页、播放模式切换等能力。

## 2. 三期目标与范围

### 2.1 目标

- 让宝宝可以独立观看绘本，系统自动朗读文字并翻页
- 支持家长陪伴模式，手动控制翻页和朗读
- 复用现有收藏、最近播放、评分等能力
- 提供简单易用的绘本管理能力

### 2.2 范围

- 绘本资源管理
- 绘本页面管理
- 绘本阅读页面
- 自动朗读与自动翻页
- 播放模式切换（自动/手动）
- 阅读进度记忆
- 与现有功能融合（收藏、最近播放、评分）

### 2.3 非范围

- 绘本内容自动生成
- AI 绘本创作
- 多语言版本切换
- 绘本社区与分享
- 绘本互动游戏
- 复杂动画效果

## 3. 与前期现状的衔接

### 3.1 一期、二期已具备能力

- 前台首页支持音频、视频资源浏览、搜索、筛选、排序
- 音频支持底部抽屉播放器，支持播放列表与播放模式
- 视频支持独立播放页
- 资源支持点击统计与匿名评分
- 收藏功能、最近播放功能
- 管理侧支持资源列表、资源新增编辑、上下架、异常标记、链接测试

### 3.2 三期扩展点

- 新增绘本资源类型，在首页列表中与音频、视频混合展示
- 新增绘本阅读页面，支持图片展示、文字朗读、自动翻页
- 管理端新增绘本维护页面，支持逐页上传图片和文字
- 绘本复用现有收藏、最近播放、评分能力
- 新增阅读进度记忆，记录看到第几页

### 3.3 三期技术取舍

- 绘本采用独立数据表，不与 `media_resource` 合并
- 文字转语音使用阿里云 TTS，提供高质量儿童音色
- 阅读进度存 Redis，与音频播放器状态类似
- 图片资源优先考虑对象存储 + CDN，保证加载速度
- 三期不支持预生成音频，全部实时调用 TTS API

## 4. 总体方案设计

### 4.1 绘本数据结构

绘本采用"一对多"结构：

- 一个绘本包含多个页面
- 每个页面包含一张图片和一段文字
- 页面按页码顺序排列

### 4.2 文字转语音方案

三期采用阿里云 TTS 实时合成方案：

**技术选型：阿里云语音合成（TTS）**

- 后端调用阿里云 TTS API 实时生成语音
- 返回音频流或临时音频 URL 给前端播放
- 支持儿童音色，音质优于浏览器原生 API

**成本说明：**

- 付费模式：按字符数收费，约 0.0025 元/次（每次最多 300 字符）
- 免费额度：每月前 100 万字符免费
- 成本估算：假设一个绘本 10 页，每页 50 字，共 500 字，单次阅读成本约 0.005 元
- 在免费额度内（100 万字符/月 = 2000 本绘本），成本为 0

**实现方式：**

- 前端请求后端接口，传入文字内容
- 后端调用阿里云 TTS API 生成音频
- 返回音频 URL 或 Base64 音频数据
- 前端使用 HTML5 Audio 播放

**优点：**

- 音质优秀，支持多种儿童音色
- 语速、音调可调节
- 稳定性高，不受浏览器兼容性影响
- 免费额度充足，初期成本可控

**缺点：**

- 依赖网络，首次加载可能有延迟
- 超出免费额度后产生费用

### 4.3 自动翻页逻辑

核心流程：

1. 展示当前页图片
2. 开始朗读当前页文字
3. 朗读结束后，延迟 1-2 秒（让宝宝看图）
4. 自动翻到下一页
5. 重复直到最后一页

### 4.4 播放模式设计

支持两种模式：

- 自动播放模式（默认）：自动朗读 + 自动翻页，适合宝宝独立观看
- 手动翻页模式：显示图片和文字，家长可点击"朗读"按钮触发朗读，手动左右滑动翻页，适合家长陪伴讲解

### 4.5 防误触设计

为避免宝宝在自动播放模式下误触翻页，提供以下防护机制：

- 锁定模式开关：在自动播放模式下，可开启"锁定翻页"功能
- 开启锁定后，禁用手动滑动翻页和上一页/下一页按钮
- 仅保留暂停/播放按钮和锁定开关本身
- 锁定状态在阅读进度中记忆，下次打开保持
- 锁定开关位置建议：顶部右上角，图标为锁形状，开启时高亮显示

### 4.6 阅读进度记忆

- 记录当前绘本看到第几页
- 记录播放模式（自动/手动）
- 记录锁定状态（是否开启防误触）
- 下次打开时自动恢复到上次阅读位置和设置
- 进度存储在 Redis 中，与音频播放器状态类似

## 5. 绘本阅读功能详细设计

### 5.1 业务目标

让宝宝可以通过图片和语音朗读的方式体验绘本内容，支持自动播放和家长陪伴两种模式。

### 5.2 交互设计

绘本阅读页面结构：

- 主区域：绘本图片（全屏或留白）
- 底部区域：
  - 当前页文字（可选显示/隐藏）
  - 进度指示器（如 3/10）
  - 控制按钮：上一页、播放/暂停、下一页
  - 模式切换：自动/手动
- 顶部：
  - 左侧：返回按钮
  - 右侧：收藏按钮、锁定开关（自动模式下显示）

### 5.3 自动播放模式

- 进入页面后自动开始朗读第一页
- 朗读结束后延迟 1-2 秒自动翻到下一页
- 最后一页朗读结束后停止，不自动循环

### 5.4 手动翻页模式

- 进入页面后不自动朗读
- 显示图片和文字
- 家长可点击"朗读"按钮触发当前页朗读
- 手动左右滑动翻页

### 5.5 用户干预规则

- 手动点击翻页时，停止当前朗读，立即切换到目标页
- 暂停按钮：暂停朗读和自动翻页
- 重播按钮：从第一页重新开始
- 切换模式时，保持当前页不变

### 5.6 锁定模式规则

- 锁定模式仅在自动播放模式下可用
- 开启锁定后：
  - 禁用手动滑动翻页
  - 禁用上一页/下一页按钮（按钮置灰或隐藏）
  - 保留暂停/播放按钮
  - 保留锁定开关本身
  - 保留返回按钮和收藏按钮
- 关闭锁定后：
  - 恢复所有翻页控制
- 锁定状态随阅读进度一起保存到 Redis

### 5.7 关键体验点

- 图片预加载：至少预加载下一页图片，避免翻页时白屏
- 音频预加载：提前调用 TTS API 生成下一页音频
- 加载状态：首次加载时显示 loading，避免空白
- 网络异常处理：TTS 失败时显示文字，提示"朗读功能暂不可用"
- 翻页动画：左右滑动、淡入淡出或翻书效果
- 锁定提示：首次开启锁定时，显示轻提示"已锁定翻页，防止误触"

### 5.8 朗读时长计算

- 阿里云 TTS：通过返回的音频文件 `audio.duration` 获取实际时长
- 延迟时间：固定 1-2 秒，可配置
- 超时处理：TTS 请求超时时间设置为 10 秒，超时后显示错误提示

### 5.9 阅读进度记忆

- 记录当前绘本 ID 和当前页码
- 记录播放模式（自动/手动）
- 记录锁定状态（是否开启防误触）
- 下次打开时自动恢复到上次阅读位置和设置
- 进度存储在 Redis 中，Key 为 `bbplay:book:progress:{uid}`

## 8. 数据模型设计

### 8.1 绘本资源表 `picture_book`

建议字段如下：

- `id`：主键
- `title`：正式标题
- `nickname`：自定义昵称
- `alias`：别名
- `series_id`：系列 ID
- `age_range`：适龄范围
- `cover_url`：封面链接
- `description`：简介
- `page_count`：总页数
- `is_published`：是否上架
- `is_abnormal`：是否异常
- `abnormal_remark`：异常原因
- `click_count`：点击次数
- `rating_avg`：平均评分
- `rating_count`：评分次数
- `complete_read_count`：完整阅读次数
- `sort_weight`：排序权重
- `created_at`
- `updated_at`

### 8.2 绘本页面表 `picture_book_page`

建议字段如下：

- `id`：主键
- `book_id`：关联绘本 ID
- `page_number`：页码
- `image_url`：图片地址
- `text_content`：页面文字
- `audio_url`：可选，预生成的音频地址
- `duration_sec`：朗读时长（用于自动翻页）
- `created_at`
- `updated_at`

### 8.3 关键设计决策

- 绘本采用独立表，不与 `media_resource` 合并
- 原因：绘本是"一对多"结构，与音频/视频的单资源模型差异较大
- 但可以在前台列表中混合展示，通过类型区分

### 8.4 Redis 存储设计

阅读进度 Key：`bbplay:book:progress:{uid}`

- 类型：Hash
- Field：`bookId`
- Value：JSON 字符串，包含 `currentPage`、`lastReadAt`

阅读进度数据结构 `BookProgressItem`：

- `bookId`：绘本 ID
- `currentPage`：当前页码
- `playMode`：播放模式，`AUTO` 或 `MANUAL`
- `locked`：是否锁定翻页，`true` 或 `false`
- `lastReadAt`：最近阅读时间

## 9. 前后端接口设计

### 9.1 前台接口

`GET /api/picture-books/list`

- 获取前台绘本列表
- 支持参数：
  - `keyword`
  - `ageRange`
  - `seriesId`
  - `sortBy`
  - `order`
  - `page`
  - `pageSize`
- 只返回上架且非异常绘本

`GET /api/picture-books/{id}`

- 获取单个绘本信息
- 包含所有页面数据

`GET /api/picture-books/{id}/pages`

- 获取绘本所有页面
- 按页码排序

`POST /api/picture-books/{id}/click`

- 记录点击次数

`POST /api/picture-books/{id}/rate`

- 提交评分
- 参数：`score`

`GET /api/user/book-progress/{bookId}`

- 获取阅读进度

`PUT /api/user/book-progress/{bookId}`

- 更新阅读进度
- 请求体：
  - `currentPage`：当前页码
  - `playMode`：播放模式（可选）
  - `locked`：锁定状态（可选）

`POST /api/tts/synthesize`

- 文字转语音
- 请求体：
  - `text`：要合成的文字
  - `voice`：音色（可选，默认儿童音色）
  - `speed`：语速（可选，默认 1.0）
- 返回：
  - `audioUrl`：音频临时 URL
  - `duration`：音频时长（秒）

### 9.2 管理接口

`GET /api/admin/picture-books/list`

- 获取绘本管理列表

`POST /api/admin/picture-books`

- 新增绘本

`PUT /api/admin/picture-books/{id}`

- 编辑绘本

`DELETE /api/admin/picture-books/{id}`

- 删除绘本

`PATCH /api/admin/picture-books/{id}/publish`

- 上下架绘本

`PATCH /api/admin/picture-books/{id}/abnormal`

- 标记或取消异常

`GET /api/admin/picture-books/{id}/pages`

- 获取绘本所有页面

`POST /api/admin/picture-books/{id}/pages`

- 新增页面

`PUT /api/admin/picture-books/{id}/pages/{pageId}`

- 编辑页面

`DELETE /api/admin/picture-books/{id}/pages/{pageId}`

- 删除页面

`PUT /api/admin/picture-books/{id}/pages/reorder`

- 调整页面顺序
- 请求体：`pageIds` 数组

### 11.4 TTS 实现

使用阿里云 TTS API：

```javascript
// 调用后端接口获取音频
const response = await fetch('/api/tts/synthesize', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ text: pageText })
})
const { audioUrl, duration } = await response.json()

// 播放音频
const audio = new Audio(audioUrl)
audio.play()

// 监听事件
audio.onended = () => {
  // 朗读结束，触发自动翻页
}
audio.onerror = () => {
  // 朗读失败，显示错误提示
}
```

**音频缓存策略：**

- 使用 Map 缓存已生成的音频 URL
- Key 为页面文字内容的 hash
- 同一绘本二次打开时直接使用缓存
- 页面卸载时清理缓存

### 11.5 锁定模式实现

```javascript
const locked = ref(false) // 锁定状态
const playMode = ref('AUTO') // 播放模式

// 开启锁定
const toggleLock = () => {
  locked.value = !locked.value
  // 保存到阅读进度
  saveProgress()
}

// 翻页控制
const canSwipe = computed(() => {
  return playMode.value === 'MANUAL' || !locked.value
})

// 按钮禁用状态
const prevDisabled = computed(() => {
  return playMode.value === 'AUTO' && locked.value
})
const nextDisabled = computed(() => {
  return playMode.value === 'AUTO' && locked.value
})
```

## 12. 潜在问题和建议

### 12.1 问题 1：TTS 成本

- 阿里云 TTS 按字符收费，每月前 100 万字符免费
- 成本估算：假设一个绘本 10 页，每页 50 字，共 500 字，单次阅读成本约 0.005 元
- 在免费额度内（100 万字符/月 = 2000 本绘本），成本为 0
- 建议：初期使用免费额度，后续根据实际使用量评估成本

### 12.2 问题 2：图片加载速度

- 路由插件内网穿透方案可能影响图片加载速度
- 建议：绘本图片优先考虑对象存储 + CDN，保证流畅体验

### 12.3 问题 3：文字与图片对应关系

- 需要明确：一页一段文字，还是一页可以有多段文字？
- 建议：三期简化为一页一段文字

### 12.4 问题 4：多语言支持

- 是否需要支持中英文切换？
- TTS 需要指定语言参数
- 建议：三期仅支持中文，后续扩展

### 12.5 问题 5：音频加载延迟

- 阿里云 TTS 实时合成可能有 1-2 秒延迟
- 建议：提前预加载下一页音频，减少等待时间
- 首次加载时显示 loading 状态

### 12.6 问题 6：误触防护

- 宝宝可能在自动播放时误触翻页
- 建议：提供锁定模式，禁用手动翻页
- 锁定状态随阅读进度保存，下次打开保持

## 16. 三期结论

三期采用独立绘本数据模型 + 实时 TTS 方案后，系统将具备完整的绘本阅读能力：

- 宝宝可以独立观看绘本，系统自动朗读并翻页
- 家长可以陪伴讲解，手动控制翻页和朗读
- 绘本与音频、视频在首页统一展示，体验一致
- 复用现有收藏、最近播放、评分等能力，降低开发成本

该方案与当前项目代码结构兼容，新增范围明确，适合直接进入开发实施。

核心技术点：

- 数据结构：绘本 + 页面的一对多关系
- TTS 方案：优先使用浏览器原生 API，降低成本
- 自动翻页：基于音频时长 + 固定延迟
- 用户体验：支持自动和手动两种模式
- 图片加载：优先考虑对象存储 + CDN

建议先实现 MVP 版本验证核心流程，特别是 TTS 的音质和加载速度是否满足老人和宝宝的使用需求。
